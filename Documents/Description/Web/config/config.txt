Назначение файла:
Файл config.json используется для централизованного хранения настроек приложения. 
Это позволяет легко изменять параметры (например, адрес сервера или данные для подключения к базе данных)
без необходимости редактирования исходного кода приложения.

Структура файла:
    Файл разделен на две основные секции:
        web — настройки, связанные с веб-приложением.
        db — настройки, связанные с базой данных.

    1. Секция web:
        Эта секция содержит параметры, связанные с веб-сервером и работой приложения.
        session_timeout (число): Время жизни сессии пользователя в секундах.
        host (строка): IP-адрес или доменное имя сервера, на котором развернуто веб-приложение.
        domain (строка):nДомен, на котором работает приложение.
        generator_url (строка):URL-адрес для генерации GUID (глобально уникальных идентификаторов).
            Используется http://domain/generator.php. 
            Указывает на скрипт, который генерирует GUID для различных объектов в системе (например, для пользователей или оборудования).

    2. Секция db:
        Эта секция содержит параметры для подключения к базе данных.
        host (строка): IP-адрес или доменное имя сервера базы данных.
        port (строка): Порт, на котором работает база данных.
        name (строка):Имя базы данных, к которой нужно подключиться.
        user (строка): Имя пользователя для подключения к базе данных.
        password (строка):тПароль для подключения к базе данных.

{
    "web": {
        "session_timeout": 3600,
        "host": "",
        "domain": "domain",
        "generator_url": "http://domain/generator.php"
    },
    "db": {
        "host": "",
        "port": "5432",
        "name": "name",
        "user": "user",
        "password": "password"
    }
}
