Файл input_err.js содержит JavaScript-код, который реализует базовую проверку 
валидности полей ввода на странице авторизации (входа в систему). Этот скрипт 
предотвращает отправку формы, если обязательные поля не заполнены, и добавляет 
визуальные индикаторы для невалидных полей. 

    1. Ожидание загрузки DOM
            document.addEventListener("DOMContentLoaded", function () { ... }):
        Этот код гарантирует, что скрипт выполнится только после того,
        как весь HTML-документ будет полностью загружен и готов к взаимодействию.
        Это важно, потому что скрипт взаимодействует с элементами DOM 
        (формой и полями ввода), которые должны быть доступны на момент выполнения кода.

    2. Получение элементов DOM
            const form = document.querySelector(".authorization form");
            const inputs = document.querySelectorAll(".authorization input[required]");
            form:
        Получает ссылку на форму внутри элемента с классом .authorization с помощью метода document.querySelector.
        Эта форма будет использоваться для проверки валидности полей перед отправкой.
    
            inputs:
        Получает все обязательные поля ввода (input[required]) внутри 
        элемента с классом .authorization с помощью метода document.querySelectorAll.
        Эти поля будут проверяться на заполненность.

    3. Добавление обработчика события на форму
            form.addEventListener("submit", function (event) {
                let isFormValid = true;

                inputs.forEach(input => {
                    if (!input.value.trim()) {
                        input.classList.add("invalid"); // Добавляем класс для невалидного поля
                        isFormValid = false;
                    } else {
                        input.classList.remove("invalid"); // Убираем класс, если поле валидно
                    }
                });

                if (!isFormValid) {
                    event.preventDefault(); // Останавливаем отправку формы, если есть ошибки
                }
            });
            form.addEventListener("submit", function (event) { ... }):

        Добавляет обработчик события submit на форму.
        Когда пользователь пытается отправить форму, выполняется функция, 
        которая проверяет валидность полей.

            let isFormValid = true:
        Переменная, которая хранит состояние валидности формы. Изначально предполагается, что форма валидна.
        
        Цикл forEach:
        Проходит по всем обязательным полям ввода (inputs).
            if (!input.value.trim()):
        Проверяет, пусто ли поле ввода (после удаления пробелов с помощью trim()).
        Если поле пустое, добавляет класс invalid к полю и устанавливает isFormValid в false.
            else:
        Если поле заполнено, удаляет класс invalid.
            if (!isFormValid):
        Если хотя бы одно поле не заполнено, форма считается невалидной.
            event.preventDefault():
        Отменяет стандартное поведение формы (отправку данных), если форма невалидна.

    4. Как работает скрипт
        Инициализация:
            После загрузки страницы скрипт находит форму и все обязательные поля ввода.
        Проверка валидности:
            При попытке отправить форму скрипт проверяет, заполнены ли все обязательные поля.
            Если поле пустое, к нему добавляется класс invalid, и форма не отправляется.
        Визуальная обратная связь:
            Класс invalid может быть использован для добавления стилей к невалидным полям (например, красная рамка).
