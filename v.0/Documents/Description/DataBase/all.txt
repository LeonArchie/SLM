База данных, описанная в предоставленных файлах, представляет собой систему управления оборудованием, 
пользователями и их привилегиями. Она предназначена для контроля доступа к оборудованию, управления 
учетными записями пользователей, распределения ролей и привилегий, а также отслеживания статусов 
различных объектов (оборудование, инциденты, задачи, запросы).

Основные сущности базы данных:

Оборудование (Hardware):
    Хранит информацию о физических устройствах (серверы, компьютеры и т.д.).
    Основные атрибуты: уникальный идентификатор (hwid), название, статус, домен, IP/MAC-адрес, SSH-ключ, операционная система.

Пользователи (Users):
    Содержит данные о зарегистрированных пользователях системы.
    Основные атрибуты: уникальный идентификатор (userid), логин, хэш пароля, имя для отображения, электронная почта, роль пользователя (roleid), дата регистрации.

Роли (Roles):
    Определяет роли пользователей в системе (например, администратор, пользователь, гость).
    Основные атрибуты: уникальный идентификатор (roleid), название роли.

Привилегии (Privileges):
    Определяет, какие действия могут выполнять пользователи с определенными ролями.
    Основные атрибуты: идентификатор роли (roleid), флаги для различных действий (регистрация, просмотр и редактирование учетных записей, управление оборудованием, создание задач и т.д.).

Статусы (Statuses):
    Хранит возможные статусы для объектов системы (оборудование, инциденты, задачи, запросы).
    Основные атрибуты: уникальный идентификатор (stid), название статуса, флаги для применения статуса к различным объектам.

Привилегии доступа к оборудованию (Hardware User):
    Управляет доступом пользователей к конкретному оборудованию.
    Основные атрибуты: уникальный идентификатор записи (id), идентификатор оборудования (hwid), идентификатор пользователя (userid), роль доступа (редактирование, просмотр, доступ закрыт).

Связи между таблицами:
    Пользователи и Роли:
        Каждый пользователь (users.txt) связан с ролью (name_rol.txt) через поле roleid.
        Роль определяет, какие привилегии (privileges.txt) имеет пользователь.

    Привилегии и Роли:
        Каждая роль (name_rol.txt) имеет набор привилегий (privileges.txt), которые определяют, какие действия может выполнять пользователь с этой ролью.

    Оборудование и Пользователи:
        Доступ пользователей к оборудованию управляется через таблицу hardvare_user.txt, которая связывает пользователей (users.txt) и оборудование (hardvare.txt).

    Статусы:
        Статусы (status.txt) могут применяться к различным объектам (оборудование, инциденты, задачи, запросы) в зависимости от флагов (hardvare, incident, tack, request).

Основные функции базы данных:

    Управление оборудованием:
        Хранение информации о физических устройствах.
        Управление доступом пользователей к оборудованию через таблицу hardvare_user.txt.

    Управление пользователями:
        Регистрация и хранение данных пользователей.
        Назначение ролей и привилегий для контроля доступа к различным функциям системы.

    Управление ролями и привилегиями:
        Определение ролей и их прав доступа к различным разделам системы (оборудование, учетные записи, задачи, инциденты).

    Управление статусами:
        Отслеживание состояния объектов (оборудование, инциденты, задачи, запросы) через систему статусов.


Пример использования:
    Регистрация пользователя:
        Пользователь регистрируется в системе, его данные сохраняются в таблице users.txt.
        Ему назначается роль (roleid), которая определяет его привилегии (privileges.txt).

Доступ к оборудованию:
        Пользователь может получить доступ к оборудованию, если ему назначены соответствующие права в таблице hardvare_user.txt.
        Роль пользователя определяет, может ли он только просматривать оборудование или также редактировать его.

Создание задачи:
        Пользователь с ролью, у которой установлен флаг createtask, может создавать задачи.
        Статус задачи может быть изменен в зависимости от флагов в таблице status.txt.
